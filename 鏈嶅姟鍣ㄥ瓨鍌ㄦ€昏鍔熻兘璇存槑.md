# CloudMind 服务器存储总览功能说明

本次新增的是管理员端“服务器存储”功能，用于让管理者快速了解当前服务器/宿主机磁盘容量、业务文件实际占用、用户配额分配和回收站占用情况。

## 后端新增接口

### GET /api/admin/storage/overview

请求头：

```http
X-Token: 管理员登录 token
```

返回内容包括：

- `serverTotalBytes`：Spring Boot 应用所在服务器磁盘总容量
- `serverUsedBytes`：服务器磁盘已用容量
- `serverUsableBytes`：服务器磁盘可用容量
- `serverUsedPercent`：服务器磁盘使用率
- `serverUsablePercent`：服务器磁盘可用率
- `activeFileBytes`：CloudMind 未删除文件的实际业务占用
- `deletedFileBytes`：回收站文件占用
- `quotaAllocatedBytes`：所有用户配额合计
- `quotaOversoldBytes`：用户配额合计超过服务器总容量时的超分风险
- `diskStores`：服务器所有可读取磁盘/分区明细

## 前端新增位置

登录管理员账号后进入：

```text
管理后台 -> 服务器存储
```

页面展示：

1. 服务器总容量
2. 服务器已用容量
3. 服务器可用容量
4. 全站文件实际占用
5. 用户配额已分配
6. 回收站占用
7. 业务占服务器比例
8. 配额超分风险
9. 磁盘分区明细

## 注意

服务器容量读取的是 Spring Boot 应用所在服务器的文件系统。如果 MinIO 运行在 Docker Desktop 里，实际存储通常与 Docker Desktop / 宿主机磁盘相关；业务已用空间则按数据库中 CloudMind 文件记录统计。
